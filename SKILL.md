# SEO для Wildberries

## Core Rule

Собирать семантику только по наиболее похожим товарам и использовать только свойства, которые подтверждены ТЗ пользователя. Не добавлять чужие бренды, неподходящие категории, ложные свойства и повторяющиеся ключевые слова ради SEO.

## Start Gate

При каждом новом запуске сначала написать пользователю: `Заполни ТЗ` и запросить одним сообщением:

- товар;
- категория;
- пол;
- цвет;
- материал;
- фасон;
- посадка;
- сезон;
- размеры;
- комплектация;
- особенности;
- поисковый запрос;
- минимальная выручка в рублях.

Не начинать сбор, пока неизвестны поисковый запрос, порог выручки и реальные свойства товара. Пол использовать для отбора конкурентов и очистки семантики, но не включать в заголовок Wildberries.

## Required Setup

- Использовать skill `browser:control-in-app-browser` и явно выбрать внутренний браузер Codex (`iab`).
- Использовать skill `spreadsheets:Spreadsheets` для проверки и создания итогового Excel.
- Не переключаться на Яндекс Браузер или Google Chrome без прямой просьбы пользователя.
- Создать отдельную папку запуска для выгрузок и результатов.
- Не обходить защиту Wildberries. При ограничении сохранить прогресс и сообщить, где остановлена работа.

## Workflow

1. **Prepare run**
   - Создать папку `outputs/<product-slug>-<date>/wildbox-exports`.
   - Записать ТЗ, поисковый запрос и порог выручки.
   - Для финального Excel подготовить отдельную папку `~/Desktop/Codex/SEO/<поисковый запрос>`.
   - Название папки взять из поискового запроса пользователя, сохранить на русском и привести лишние пробелы к одному.
   - Заменить только недопустимые для имени папки символы `/ : * ? " < > |` на дефис.
   - Если папка с таким запросом уже содержит `Итоговое SEO.xlsx`, не перезаписывать результат: создать рядом папку `<поисковый запрос> — <ГГГГ-ММ-ДД ЧЧ-ММ>`.

2. **Open search**
   - Открыть внутренний браузер Codex.
   - Перейти на Wildberries и выполнить поиск по запросу пользователя.
   - Убедиться, что открылась выдача Wildberries по запросу пользователя.
   - Если в браузере доступны показатели Wildbox, использовать их. Если Wildbox недоступен, использовать подтвержденный пользователем рабочий способ получения выручки и запросов и явно отметить источник.

3. **Select competitors**
   - Использовать расчетную выручку Wildbox за 30 дней для всего артикула.
   - Применять условие строго `выручка > порог`.
   - Учитывать приблизительные значения Wildbox.
   - Найти и выбрать 30 наиболее похожих товаров, а не первые 30 выше порога.
   - Не ограничиваться первой страницей выдачи. Последовательно проверять страницы 1, 2, 3, 4, 5 и следующие страницы, пока не будут приняты 30 релевантных конкурентов.
   - После завершения каждой страницы переходить на следующую через видимую пагинацию Wildberries и проверять, что номер страницы или состав выдачи изменился.
   - Если на текущей странице недостаточно подходящих товаров или большинство карточек не проходит порог, продолжать поиск на следующих страницах.
   - Не делать вывод «релевантных конкурентов меньше 30», пока не проверены как минимум первые 5 страниц и все доступные следующие страницы, необходимые для набора 30 конкурентов.
   - Остановить пагинацию можно только после набора 30 принятых конкурентов, фактического окончания выдачи, повторения уже проверенной страницы или блокировки Wildberries.
   - Проверить тип товара, материал, фасон, посадку, сезон, комплектацию и визуальное сходство.
   - Исключить комплекты, другой тип товара и нерелевантные фасоны.
   - Не брать один артикул повторно.
   - Записывать каждый просмотренный артикул в `checkedArticles`, даже если он отклонен.
   - Для каждого артикула сохранять ссылку, номер страницы выдачи, выручку, статус `принят` или `отклонен` и причину решения.
   - Если после исчерпания выдачи релевантных карточек меньше 30, использовать найденные и сообщить фактическое количество, число проверенных страниц и точную причину остановки.

4. **Download queries**
   - Открывать карточки по одной.
   - Нажимать в Wildbox вкладку `Запросы`.
   - Дождаться таблицы запросов.
   - Нажимать `Скачать список` и сохранять отдельный Excel.
   - После каждого скачивания проверять, что файл существует, не пуст и содержит запрос и частотность за 30 дней.
   - Для принятых конкурентов записывать артикул, ссылку, выручку, приблизительность данных и имя файла.
   - Не считать SEO проверенным по артикулу, пока запросы этого артикула не получены или не указана причина отсутствия данных.

5. **Collect competitor characteristics**
   - В каждой принятой карточке раскрыть полный видимый блок характеристик товара.
   - Сохранить по артикулу пары `поле — значение`; не смешивать данные разных артикулов.
   - Нормализовать только очевидные варианты написания названий полей, не объединять разные по смыслу характеристики.
   - Для каждого поля посчитать число уникальных конкурентов, которые его заполнили, и сохранить встречавшиеся значения.
   - Отсутствие поля в карточке считать `не показано/не заполнено`, а не доказательством отсутствия свойства у товара.
   - Не переносить значение конкурента в нашу карточку без подтверждения ТЗ пользователя.
   - Если доступен полный перечень полей категории из кабинета продавца, шаблона категории или официальной схемы Wildberries, сохранить его отдельно и сопоставить с наблюдаемыми полями конкурентов.
   - Если полный перечень недоступен, прямо указать, что собран список наблюдаемых характеристик конкурентов, а не исчерпывающая схема категории.

6. **Merge exports**
   - Запустить `scripts/merge_wildbox_exports.mjs` для папки выгрузок.
   - Привести запросы к нижнему регистру и удалить лишние пробелы.
   - Полностью одинаковые запросы оставить один раз.
   - Частотность одинаковых запросов не суммировать; сохранить максимальное доступное значение.
   - Посчитать, у скольких конкурентов встречается запрос.
   - Отсортировать по частотности за 30 дней от большей к меньшей.

7. **Clean semantics**
   - Прочитать `references/semantic-cleaning.md`.
   - Разделить запросы на `релевантные`, `сомнительные` и `исключенные`.
   - Для каждого исключенного запроса указать причину.
   - Не удалять сомнительные запросы молча: оставить их для ручной проверки.

8. **Create SEO**
   - Прочитать `references/wb-content-rules.md`.
   - Подготовить одно окончательное название длиной не более 60 символов.
   - Не использовать бренд, пол, возраст и сезон в заголовке.
   - Строить заголовок по схеме: `тип товара + ключевой фасон/свойство + цвет`.
   - Использовать не более трех высокочастотных фраз как основу, без повторений слов. Полное совпадение фразы необязательно.
   - Предложить ключи для характеристик только при соответствии реальному товару.
   - Для каждой наблюдаемой или официальной характеристики назначить решение: `Заполнить`, `Уточнить` или `Не использовать`.
   - В `Заполнить` помещать только поля со значением, подтвержденным ТЗ.
   - В `Уточнить` помещать важные для категории поля, по которым у пользователя нет данных; сформулировать конкретный вопрос.
   - В `Не использовать` помещать нерелевантные поля и неподтвержденные значения конкурентов с причиной.
   - Написать красивое продающее описание целевым объемом 900–1500 символов, если лимит категории Wildberries не ниже.
   - Естественно использовать 8–15 наиболее релевантных высоко- и среднечастотных ключей.
   - Не вставлять ключи списком, не повторять одинаковые фразы и не придумывать свойства товара.
   - Разделить описание на короткие смысловые абзацы: товар и образ, посадка и материал, преимущества, сценарии использования, сочетания и комплектация.

9. **Build final Excel**
   - Подготовить JSON по схеме `references/output-schema.md`.
   - Запустить `scripts/build_final_workbook.mjs`.
   - Создать лист `Итоговое SEO` с блоками: ТЗ, все проверенные артикулы, принятые конкуренты, анализ характеристик конкурентов, SEO-ядро, сомнительные и исключенные запросы.
   - Создать второй лист `Карточка товара`.
   - На втором листе дать одно финальное название товара, самостоятельно составленное по ТЗ и очищенному ядру.
   - Ниже разместить готовые характеристики товара.
   - После готовых характеристик разместить блок `Нужно уточнить` с важными незаполненными полями и вопросами пользователю.
   - Ниже разместить готовое естественное описание, составленное с использованием релевантных ключей без переспама.
   - Сохранить финальный файл как `~/Desktop/Codex/SEO/<поисковый запрос>/Итоговое SEO.xlsx`.
   - Рабочие JSON, изображения проверки и выгрузки Wildbox не копировать в эту папку: в пользовательской папке должен лежать только готовый Excel.

10. **Deliver**
   - Передать итоговый Excel.
   - Дать пользователю кликабельную ссылку на Excel из папки `~/Desktop/Codex/SEO/<поисковый запрос>`.
   - Кратко сообщить количество обработанных конкурентов, число релевантных запросов и все ограничения сбора.

## Recovery Rules

- Если вкладка внутреннего браузера закрылась или устарела, получить новую вкладку в той же браузерной сессии.
- Если показатели Wildbox недоступны, не придумывать выручку и попросить выгрузку либо использовать согласованный источник.
- Если карточка не открылась или выгрузка не скачалась, повторить один раз и перейти к следующему релевантному конкуренту.
- Если переход на следующую страницу не сработал, повторить один раз, получить свежую вкладку в той же сессии и продолжить с сохраненного номера страницы.
- Если страница содержит только уже проверенные артикулы, сверить номер страницы и URL; при подтвержденном повторении считать выдачу завершенной и записать это как причину остановки.
- Сохранять прогресс после каждого артикула.
- При блокировке Wildberries остановиться без обхода защиты и сохранить уже полученные файлы.
- При повторном запуске пропускать артикулы с проверенной выгрузкой.

## Quality Gate

Перед выдачей проверить:

- получены выгрузки от 30 разных релевантных конкурентов;
- если конкурентов меньше 30, проверены как минимум первые 5 страниц и все доступные следующие страницы до фактического окончания выдачи, повторения страницы или блокировки;
- в результате указаны номера всех проверенных страниц и точная причина остановки пагинации;
- в итоговом Excel есть список всех проверенных артикулов, включая отклоненные;
- у каждого проверенного артикула указаны статус и причина решения;
- все конкуренты имеют выручку строго выше порога;
- дубли запросов удалены, частотности не суммированы;
- сортировка выполнена по частотности за 30 дней;
- у каждого запроса указано число конкурентов;
- исключенные запросы имеют причины;
- заголовки не длиннее 60 символов и не содержат бренда, пола, возраста, сезона, повторов или спецсимволов;
- характеристики не противоречат ТЗ;
- характеристики конкурентов собраны по уникальным артикулам и имеют счетчик встречаемости;
- значения конкурентов не перенесены в нашу карточку без подтверждения;
- указано, является ли перечень характеристик полным для категории или только наблюдаемым у конкурентов;
- все важные неподтвержденные поля вынесены в блок `Нужно уточнить`;
- описание правдивое, естественное и без переспама;
- описание продающее, раскрывает выгоды и сценарии использования, а не состоит из двух коротких предложений;
- целевой объем описания — 900–1500 символов или максимум, разрешенный категорией;
- в описание естественно включено 8–15 релевантных ключей без списка и повторов;
- итоговый файл содержит ровно два листа: `Итоговое SEO` и `Карточка товара`;
- финальный Excel сохранен в отдельной папке внутри `~/Desktop/Codex/SEO`, названной по поисковому запросу;
- существующий результат с тем же запросом не перезаписан;
- лист `Карточка товара` содержит одно финальное название, характеристики и описание;
- финальное название не длиннее 60 символов и соответствует правилам Wildberries;
- сбор выполнен через внутренний браузер Codex.
